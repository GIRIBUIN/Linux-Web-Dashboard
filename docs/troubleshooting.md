# Troubleshooting Log

> 나중에 다시 도움이 될 만한 문제만 기록합니다.
> 증상, 원인, 해결 방법, 재발 방지 방법 중심으로 작성합니다.

---

## Writing Rule

다음 경우에 작성:
- 실제 디버깅 이슈가 발생했을 때
- 개발 환경 설정에 실패했을 때
- 파싱 로직이 예상과 다르게 동작했을 때
- API, 서비스, 실행 환경에서 이상 동작이 있었을 때
- 나중에 같은 문제가 다시 발생할 가능성이 있을 때

다음 내용은 굳이 작성하지 않아도 됨:
- 기술적 의미가 없는 일시적인 혼란
- 다시 참고할 가치가 없는 단순 실수
